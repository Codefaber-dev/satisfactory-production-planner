## §G — Goal

- Extraction data (satisfactory.wiki.gg): Miner Mk.1/Mk.2/Mk.3 = 60/120/240 items/min @100% on a normal node; power 5/15/45 MW. Purity multiplier: impure ×0.5, normal ×1, pure ×2. - Converter ore-conversion recipes already seeded (UpdateOneZero Converter block, T68): raw ore produced from another raw + Reanimated SAM — these are the "convert" source option for raw ingredients (V61).
- AWESOME Sink points (satisfactory.wiki.gg): each item has a sink point value; recyclable points/min = qty/min × points. Only **solid** items are sinkable — unpackaged fluids/gases (`is_liquid` true) are NOT sinkable (their Packaged forms are sinkable items); a few items have no sink value → 0, not recycled. No points data currently in DB → new `sink_points` column needed (T108).
- Unpackage recipes seeded (Packager) for all 12 fluids/gases that have a packaged form (Recipe enum `UNPACKAGE_*` / Ingredient enum `PACKAGED_*`): Alumina Solution, Fuel, Heavy Oil Residue, Ionized Fuel, Liquid Biofuel, Nitric Acid, Nitrogen Gas, Crude Oil (Oil), Rocket Fuel, Sulfuric Acid, Turbofuel, Water. Each consumes the Packaged form, returns the empty container (Empty Canister / Empty Fluid Tank) as byproduct. Of these only Water/Crude Oil/Nitrogen Gas are raw (tier-1); the other 9 are produced intermediates. These back the "unpackage" option (V63).

- **V58** Circular recipe dependencies resolved by linear solve, not by forcing a substitute recipe. **Two-stage detection.** Stage 1 (static, precomputed): build the directed graph over the *full* recipe catalog (incl. alts, edge product→ingredient per recipe), run Tarjan SCC → candidate loop clusters, each tagged with the `(product, recipe)` picks that enable its back-edges; cache it, invalidated on Recipe/Ingredient save (same hook as T35 — B17). Stage 2 (per plan): a loop is *active* iff the plan's selected recipes cover a candidate cluster's enabling set (cheap intersection) — no dynamic mid-walk cycle-catching, no recursion-depth probing. For each active SCC (e.g. Recycled Plastic ⇄ Recycled Rubber) build the square balance matrix A (per-product net = recipe outputs − inputs across the SCC's selected recipes) and solve `A·x = demand` exactly (RREF/matrix inverse) for recipe run-rates x — **keeping the user's selected recipes**. Detection + solve must honor the same raw/import boundaries as the calc (an imported node cuts the loop). Acyclic remainder still resolved by the topological walk (current recursive path = back-substitution for a DAG). Removes the forced-recipe swap (`useCompatibleRecipe`, CalculatesSteps.php:85) and the hard-coded name overrides (`overrideFavoritesIfNecessary`, Setters.php:132 — Recycled Rubber/Plastic, Packaged Fuel). Warn + fall back only when a loop is genuinely unsolvable: `det(A)=0` / non-productive (no net output, negative gross) — never silently swap a valid loop. Same net/gross math as V55/V56. ProductionTree TopSort cut-as-import (ProductionTree.php:72) may remain for build-ordering only; quantities authoritative from the solver.
- **V57** Issue #4 restore guarantee (strengthens V32; the half T73 left untested): reloading a saved factory's Plan URL reconstructs *identical* recipe selections — proven by a round-trip test that drives the controller (`/dashboard/...?choices[...]` → `ProductionController::show`/`multi` + `baseData`) and asserts the re-derived selection state, NOT just the save payload (T73's test covered only the latter — cf B32). Restore resolution must not silently collapse a stored non-default pick to its base recipe: `Recipe::ofName` returning a base-recipe fallback for an unmatched description, then `baseData` (ProductionController:29) default-rejecting it, must never revert a user choice.
- **V56** Perfect Ratio Factory mode: plan-level action scaling all output yields by smallest factor S s.t. every recipe step's building count is a whole integer at 100% clock (no fractional/underclocked machine anywhere). `S = LCM(denominators of all steps' exact fractional machine demand)` (rational machine counts). Reuses ScaleOutputs (one-shot scalar on every output yield, output ratios preserved); mutual loops covered — integer machine-vector falls out of the joint LCM. Needs exact fractional machine demand per step (expose from backend if the displayed count is already rounded). Guard: if S — or any resulting building count — exceeds cap → warn + do not apply (offer nearest feasible); factory could be impractically large. Not a persistent param — recompute on demand like Scale Up.
